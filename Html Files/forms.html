Input Element

The <input> element is the most important form element.

The <input> element can be displayed in several ways, depending on the type attribute.

Some examples are
TYPE DESCRIPTION
<input type="text"> Single line text input
<input type="radio"> Defines radio button to select one of many choices
<input type="checkbox"> Allows the user to select one or more options
<input type="password"> Characters in a a passwordfield are masked
<input type="submit"> Defines a submit button for submiting the form

Text Input

<input type="text"> defines a one-line input field for text input:


Example 1:

<form>
First name:
<input type="text" name="firstname">
Last name:
<input type="text" name="lastname">
</form>

Radio Button Input

<input type="radio"> defines a radio button.
Radio buttons let a user select ONE of a limited number of choices:


Example 2:

<form>
<input type="radio" name="gender" value="male"> Male
<input type="radio" name="gender" value="female"> Female
<input type="radio" name="gender" value="other"> Other
</form>

Checkbox

Checkboxes allows the user to select one or more option from a pre-defined set of options. It is created using an <input> element whose type attribute has a value of checkbox

Example 3:

< form >
<input type="checkbox" name="sports" id="soccer">
<input type="checkbox" name="sports" id="cricket">
<input type="checkbox" name="sports" id="baseball">
</form>

Password

This is also a single-line text input controls created using an <input> element whose type attribute has a value of password. Here's an example of a single-line password input used to take user password:

Example 4:

< form >
<input type="password" name="user-password" id="user-pwd">
</form>

Submit and Reset Button

A submit button is used to send the form data to a web server. When submit button is clicked the form data is sent to the file specified in the form's action attribute to process the submitted data. A reset button resets all the forms control to default values.


Example 5:

<input type="text" name="first-name" id="first-name">
<input type="submit" value="Submit">
<input type="reset" value="Reset">
<form>


When to use GET and POST?

The default method when submitting form data is GET. However, when GET is used, the submitted form data will be visible in the page address field:

Always use POST if the form data contains sensitive or personal information. The POST method does not display the submitted form data in the page address field.
